# AI Agent с многоуровневой памятью и стратегиями контекста

Проект представляет собой интеллектуального агента, использующего Yandex GPT API, с поддержкой трёх слоёв памяти и трёх стратегий управления контекстом. Агент предназначен для персонализированного диалога с учётом профиля пользователя, текущей задачи и истории взаимодействия.

## Архитектура

### Три слоя памяти
1. **Краткосрочная память (ShortTermMemory)** – хранит последние N сообщений диалога (скользящее окно).
2. **Рабочая память (WorkingMemory)** – структурированные данные текущей задачи (ключ-значение).
3. **Долговременная память (LongTermMemory)** – профиль пользователя (роль, уровень экспертизы, предпочтения) и важные факты, извлечённые из диалога.

### Три стратегии управления контекстом
1. **Sliding Window** – классическое скользящее окно последних сообщений.
2. **Sticky Facts** – автоматически извлекает и сохраняет важные факты из диалога, добавляя их в системный промпт.
3. **Branching** – позволяет создавать параллельные ветки диалога и переключаться между ними.

## Установка

1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd AI_Agent
   ```

2. Создайте виртуальное окружение (рекомендуется):
   ```bash
   python -m venv .venv
   .venv\Scripts\activate   # Windows
   source .venv/bin/activate # Linux/Mac
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Создайте файл `.env` в корне проекта со следующими переменными:
   ```
   YANDEX_CLOUD_FOLDER=your_folder_id
   YANDEX_CLOUD_API_KEY=your_api_key
   YANDEX_CLOUD_MODEL=yandexgpt/latest   # опционально, по умолчанию yandexgpt/latest
   ```

## Использование

Запустите агента:
```bash
python run.py
```

После запуска вы увидите список доступных команд:

```
🤖 Агент запущен. Команды:
/profile — показать текущий профиль
/set_profile поле=значение — изменить профиль (например, /set_profile preferred_style=технический)
/load_preset эксперт|новичок|менеджер — загрузить предустановленный профиль
/switch N — сменить стратегию (1=Sliding Window, 2=Sticky Facts, 3=Branching)
/window N — установить размер окна (для стратегий 1 и 2)
/facts — показать текущие факты (стратегия 2)
/branch name — создать новую ветку (стратегия 3)
/branches — список веток (стратегия 3)
/switch_branch name — переключиться на ветку (стратегия 3)
/clear — очистить краткосрочную историю (и факты для стратегии 2)
/save_working ключ: значение — сохранить в рабочую память
/save_longterm тип: содержание — сохранить в долговременную память
/show_memory — показать все три слоя памяти
/clear_memory — очистить рабочую и долговременную память
/exit — выход
```

### Пример работы
```
Вы: /load_preset эксперт
Загружен профиль: эксперт

Вы: /switch 2
Стратегия изменена на Sticky Facts

Вы: Расскажи о цифровых двойниках в промышленности.
🤖 Агент: (развёрнутый ответ с учётом профиля эксперта)
📊 Токены: prompt=450, completion=320, total=770
📈 Всего за сессию: 770
```

## Детали реализации

### Класс Agent (`agent/agent.py`)
- Инициализирует клиент Yandex GPT API через библиотеку `openai` с кастомным `base_url`.
- Управляет тремя слоями памяти и стратегиями.
- Сохраняет и загружает состояние в JSON‑файл (`data/agent_state.json`).
- Подсчитывает использованные токены.

### Стратегии (`agent/strategies.py`)
- **SlidingWindowStrategy** – передаёт в промпт последние `window_size` сообщений.
- **StickyFactsStrategy** – после каждого ответа вызывает модель для извлечения фактов и сохраняет их.
- **BranchingStrategy** – позволяет создавать независимые ветки диалога, каждая со своей историей.

### Память (`agent/memory.py`)
- **ShortTermMemory** – очередь сообщений с ограничением по длине.
- **WorkingMemory** – словарь для данных текущей задачи.
- **LongTermMemory** – хранит профиль пользователя и список фактов с временными метками.

## Возможные улучшения

1. **Добавление новых стратегий** – например, стратегии, учитывающей семантическое сходство для поиска релевантных фактов.
2. **Визуальный интерфейс** – веб‑интерфейс на Streamlit или Gradio.
3. **Интеграция с векторными БД** – для долговременной памяти вместо простого списка фактов.
4. **Поддержка других провайдеров LLM** – OpenAI, Anthropic, локальные модели.
5. **Расширенное логирование** – запись диалогов и метрик в базу данных.
